# 电子相册纯展示页面需求文档

## 1. 项目概述

### 1.1 项目背景
InkTime 项目是一个基于 ESP32 和墨水屏的电子相册系统，目前已实现了照片分析、渲染和在墨水屏上显示的功能，以及一个功能完善的网页版相册。为了提供更加简洁、专注的浏览体验，需要开发一个纯展示页面，类似于实体电子相册的纯展示部分，让用户可以专注于欣赏照片本身。

### 1.2 项目目标
开发一个极简风格的纯展示页面，仅展示照片、文字、日期和地点信息，提供类似于实体电子相册的沉浸式浏览体验，同时支持点击查看更多详细信息。

## 4. 实现方案

### 4.1 服务器端实现

#### 4.1.1 路由模块

| 路由 | 方法 | 功能描述 |
|-----|------|---------|
| /display | GET | 纯展示页面 |
| /display/<photo_id> | GET | 指定照片的纯展示页面 |
| /api/display/next | GET | 获取下一张照片的 API |
| /api/display/prev | GET | 获取上一张照片的 API |

#### 4.1.2 模板系统

- **display.html**：纯展示页面模板，仅展示核心信息

### 4.2 前端实现

#### 4.2.1 页面设计

- **纯展示页**：极简风格，背景为黑色或深色，中央展示照片，底部或侧边展示文字、日期和地点信息
- **布局**：照片占据主要区域，文字信息简洁展示，无多余元素
- **交互**：点击页面任意位置跳转到照片详情页，支持左右滑动切换照片

#### 4.2.2 交互功能

- **点击跳转**：点击页面跳转到照片详情页
- **自动切换**：定时自动切换到下一张照片（可配置）
- **手动切换**：支持左右滑动或点击屏幕边缘切换照片
- **响应式适配**：适配不同屏幕尺寸，保持照片居中显示

### 4.3 性能优化

- **图片优化**：使用适当尺寸的照片，确保加载速度
- **缓存策略**：缓存照片数据，减少 API 请求
- **代码精简**：最小化 CSS 和 JavaScript 文件大小

## 5. 界面设计

### 5.1 布局设计

- **整体布局**：全屏展示，背景为黑色或深色，照片居中显示
- **照片区域**：占据页面 70-80% 的空间，保持照片原始比例
- **信息区域**：位于照片下方或侧边，半透明背景，显示文字、日期和地点
- **交互提示**： subtle 的提示，指示用户可以点击或滑动

### 5.2 视觉风格

- **颜色**：深色背景，高对比度，突出照片本身
- **字体**：简洁、现代的无衬线字体，白色或浅灰色文字
- **动画**：平滑的过渡动画，避免突兀的效果
- **响应式**：在不同设备上保持一致的视觉体验

### 5.3 页面元素

| 元素 | 描述 | 位置 | 样式 |
|-----|------|------|------|
| 照片 | 主展示内容 | 页面中央 | 最大尺寸，保持比例 |
| 文字 | 照片描述或文案 | 照片下方 | 白色，半透明背景 |
| 日期 | 今日日期 | 文字下方或右上角 | 白色，小号字体 |
| 地点 | 拍摄地点 | 日期旁边或下方 | 白色，小号字体 |
| 切换提示 | 指示用户可以滑动或点击 | 屏幕边缘 | 半透明，仅在交互时显示 |

## 7. 部署计划

### 7.1 部署环境

- **操作系统**：Linux/Windows/MacOS
- **Python 版本**：3.8+
- **依赖项**：Flask、Bootstrap

### 7.2 部署步骤

1. 安装依赖项：`pip install -r requirements.txt`
2. 启动服务器：`python src/server/server.py`
3. 访问纯展示页：打开浏览器访问 `http://localhost:5001/display`

## 8. 风险评估

| 风险 | 影响 | 对策 |
|-----|------|------|
| 照片加载缓慢 | 影响用户体验 | 优化图片尺寸，实现预加载 |
| 自动切换干扰用户 | 影响浏览体验 | 提供开关选项，默认开启但可关闭 |
| 触摸设备交互不流畅 | 影响用户体验 | 优化触摸事件处理，确保流畅响应 |
| 浏览器兼容性问题 | 影响部分用户 | 使用标准 Web 技术，测试主流浏览器 |
