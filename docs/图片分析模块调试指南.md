# 图片分析模块调试指南

本文档提供了 `analyze_photos_docker.py` 脚本的详细调试方法，包括本地环境设置和 Docker 环境调试。

## 1. 脚本功能分析

`analyze_photos_docker.py` 是 InkTime 项目的图片分析模块，主要功能包括：

1. **扫描图片目录**：递归扫描指定目录中的图片文件
2. **过滤图片**：排除截图等低价值图片
3. **调用视觉语言模型（VLM）**：分析图片内容，生成描述、评分和类型
4. **提取 EXIF 信息**：获取图片的拍摄时间、相机信息、GPS 数据等
5. **解析城市信息**：根据 GPS 数据解析拍摄地点的城市名称
6. **生成一句话文案**：为图片生成简短的中文文案
7. **存储分析结果**：将分析结果存入 SQLite 数据库

## 2. 本地调试环境设置

### 2.1 安装依赖项

```bash
# 安装必要的 Python 库
pip install pillow requests

# 安装 exiftool（可选，但推荐）
# macOS
brew install exiftool
# Ubuntu/Debian
sudo apt-get install libimage-exiftool-perl
# CentOS/RHEL
sudo yum install perl-Image-ExifTool
```

### 2.2 准备城市数据库文件

脚本需要 `world_cities_zh.csv` 文件来解析城市信息：

1. 在 `data` 目录下创建 `world_cities_zh.csv` 文件
2. 可以从网上下载城市数据库，或使用以下示例数据：

```csv
lat,lon,name_zh,name_en
22.543096,114.057865,深圳,Shenzhen
31.230416,121.473701,上海,Shanghai
39.9042,116.4074,北京,Beijing
```

### 2.3 设置环境变量

创建 `.env` 文件，设置必要的环境变量：

```env
# 图片目录
IMAGE_DIR=/path/to/your/photos

# 数据库路径
DB_PATH=./photos.db

# VLM API URL（使用 LM Studio 本地服务）
API_URL=http://localhost:1234/v1/chat/completions

# 模型名称
MODEL_NAME=qwen3-vl-32b-instruct

# API KEY（如果需要）
API_KEY=

# 批次限制（可选）
BATCH_LIMIT=5

# 超时时间（秒）
TIMEOUT=600

# 城市数据库路径
WORLD_CITIES_CSV=./data/world_cities_zh.csv

# 家的位置（用于判断是否在本地）
HOME_LAT=22.543096
HOME_LON=114.057865
HOME_RADIUS_KM=60.0
```

### 2.4 启动 LM Studio 服务

1. 下载并安装 [LM Studio](https://lmstudio.ai/)
2. 下载支持视觉的模型，如 `qwen3-vl-32b-instruct`
3. 启动本地服务器，默认端口为 1234

## 4. Docker 环境调试

### 4.1 构建调试容器

创建 `Dockerfile.debug`：

```dockerfile
FROM python:3.10-slim

WORKDIR /app

# 安装依赖
RUN apt-get update && apt-get install -y \
    exiftool \
    && rm -rf /var/lib/apt/lists/*

COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

# 复制代码
COPY . .

# 设置环境变量
ENV IMAGE_DIR=/photos
ENV DB_PATH=/app/photos.db
ENV API_URL=http://host.docker.internal:1234/v1/chat/completions
ENV MODEL_NAME=qwen3-vl-32b-instruct
ENV WORLD_CITIES_CSV=/app/data/world_cities_zh.csv

# 运行脚本
CMD ["python", "src/analysis/analyze_photos_docker.py"]
```

### 4.2 运行调试容器

```bash
# 构建镜像
docker build -t inktime-debug -f Dockerfile.debug .

# 运行容器（挂载本地照片和 LM Studio 服务）
docker run --rm \
  -v /path/to/your/photos:/photos \
  -v $(pwd)/data:/app/data \
  --network host \
  inktime-debug
```

### 4.3 进入容器调试

```bash

  -v $(pwd)/data:/app/data \
```

## 6. 常见错误和解决方案

### 6.1 API 连接失败

**错误信息**：`LM Studio 请求失败: HTTP 404`

**解决方案**：
- 确保 LM Studio 服务正在运行
- 检查 API_URL 环境变量是否正确
- 验证模型名称是否正确

### 6.2 图片读取失败

**错误信息**：`读取图片失败：[Errno 2] No such file or directory`

**解决方案**：
- 检查 IMAGE_DIR 环境变量是否正确
- 确保图片文件存在且可访问

### 6.3 城市数据库缺失

**错误信息**：`[FATAL] 找不到城市索引文件`

**解决方案**：
- 确保 WORLD_CITIES_CSV 环境变量指向正确的文件
- 检查文件是否存在且格式正确

### 6.4 内存不足

**错误信息**：`MemoryError`

**解决方案**：
- 减小 BATCH_LIMIT 值
- 减小 VLM_MAX_LONG_EDGE 值，降低图片分辨率
- 使用更小的模型

### 6.5 数据库错误

**错误信息**：`sqlite3.OperationalError`

**解决方案**：
- 确保数据库文件路径可写
- 检查数据库表结构是否正确

## 7. 性能优化建议

1. **批量处理**：使用 BATCH_LIMIT 控制每次处理的图片数量
2. **图片缩放**：调整 VLM_MAX_LONG_EDGE 值，平衡质量和性能
3. **并行处理**：考虑使用多线程或异步处理提高效率
4. **缓存机制**：对已分析的图片建立缓存，避免重复分析

## 8. 调试命令示例

### 8.1 测试单张图片分析

创建测试脚本 `test_single_image.py`：

```python
#!/usr/bin/env python3

from pathlib import Path
import sys
import os

# 添加项目根目录到 Python 路径
sys.path.insert(0, str(Path(__file__).resolve().parent))

# 设置环境变量
os.environ['IMAGE_DIR'] = '/path/to/your/photos'
os.environ['DB_PATH'] = './test_photos.db'
os.environ['API_URL'] = 'http://localhost:1234/v1/chat/completions'
os.environ['MODEL_NAME'] = 'qwen3-vl-32b-instruct'
os.environ['WORLD_CITIES_CSV'] = './data/world_cities_zh.csv'

from src.analysis.analyze_photos_docker import call_vlm, generate_side_caption, read_exif

# 测试单张图片
image_path = Path('/path/to/your/photos/test.jpg')

print(f"分析图片: {image_path}")

# 调用 VLM 分析
try:
    result, exif_info = call_vlm(image_path)
    print("VLM 分析结果:")
    print(f"  类型: {result.get('type')}")
    print(f"  回忆分: {result.get('memory_score')}")
    print(f"  美观分: {result.get('beauty_score')}")
    print(f"  描述: {result.get('caption')}")
    print(f"  理由: {result.get('reason')}")
except Exception as e:
    print(f"VLM 分析失败: {e}")

# 生成一句话文案
try:
    caption = generate_side_caption(image_path)
    print(f"一句话文案: {caption}")
except Exception as e:
    print(f"生成文案失败: {e}")

# 读取 EXIF 信息
try:
    exif = read_exif(image_path)
    print("EXIF 信息:")
    print(f"  拍摄时间: {exif.get('datetime')}")
    print(f"  相机: {exif.get('make')} {exif.get('model')}")
    print(f"  GPS: {exif.get('gps_lat')}, {exif.get('gps_lon')}")
except Exception as e:
    print(f"读取 EXIF 失败: {e}")
```
