# ESP32 墨水屏 PCB 设计学习路线

> 目标：从零开始，最终能为 ESP32 + 微雪 7.3 寸 6 色墨水屏 (E) 设计一块完整的 PCB。
> 适合硬件零基础的软件开发者。

---

## 前置知识：理解硬件连接关系

### 为什么需要驱动板/PCB

墨水屏不能直接插到 ESP32 上，原因有三：

1. **电压不匹配**：墨水屏内部需要特殊驱动电压，ESP32 GPIO 只能输出 3.3V
2. **接口不匹配**：墨水屏排线有很多引脚（如 49 针），但实际通信只用 SPI 的 6 根信号线
3. **供电需求**：需要稳压器把电池电压（3.0-4.2V）稳定为 3.3V

驱动板/转接板的作用就是解决这三个问题。微雪的 HAT 版已经内置了驱动板，如果买裸屏则需要自己设计。

### 微雪 7.3 寸 6 色屏 (E) 关键参数

| 参数 | 值 |
|------|-----|
| 型号 | 7.3inch e-Paper HAT (E) |
| 技术 | E Ink Spectra 6 (E6) |
| 分辨率 | 800 × 480 |
| 颜色 | 黑、白、黄、红、蓝、绿（6 色） |
| 通信接口 | SPI（4 线） |
| 工作电压 | 3.3V |
| 刷新时间 | ~12 秒 |
| 灰度等级 | 2 |

### 6 色 vs 4 色：硬件上没有区别

两者的 SPI 引脚定义完全一样，区别只在软件层面（颜色编码从 4 种变成 6 种）。

| 引脚 | 功能 | 说明 |
|------|------|------|
| VCC | 3.3V 电源 | 供电 |
| GND | 电源地 | 接地 |
| DIN | SPI MOSI | 数据输入，ESP32 → 屏幕 |
| SCLK | SPI SCK | 时钟信号，ESP32 控制节拍 |
| CS | SPI 片选 | 低电平有效，选中这块屏幕 |
| DC | 数据/命令 | 高=数据，低=命令 |
| RST | 复位 | 低电平复位屏幕 |
| BUSY | 忙状态 | 屏幕输出，高=空闲，低=忙 |

### 接线方式

```
ESP32-S3 开发板          微雪 HAT 板
─────────────          ──────────
3.3V  ──────────────→  VCC
GND   ──────────────→  GND
GPIO9  ─────────────→  DIN (MOSI)
GPIO10 ─────────────→  SCLK (SCK)
GPIO11 ─────────────→  CS
GPIO12 ─────────────→  DC
GPIO13 ─────────────→  RST
GPIO14 ─────────────→  BUSY
```

## 阶段 2：电路基础知识（第 3-4 周）

### 目标
理解 PCB 上每个元件的作用，为画原理图做准备。

### 必学知识点

#### 2.1 欧姆定律
```
V = I × R

电压(V) = 电流(A) × 电阻(Ω)
```
- 3.3V 电源，10kΩ 上拉电阻，流过的电流 = 3.3V / 10000Ω = 0.33mA
- 这就是为什么上拉电阻选 10kΩ：电流够小不浪费电，又足够让信号稳定

#### 2.2 电容的作用

电容 = 微型蓄水池。在 PCB 上主要两个用途：

**滤波电容**（大容量，如 10µF、4.7µF）：
- 放在电源入口，平滑电压波动
- 类比：水管入口的大水箱，水压不稳时靠水箱缓冲

**去耦电容**（小容量，如 100nF）：
- 紧贴芯片电源引脚放置
- 芯片瞬间需要大电流时（比如 WiFi 发射），电容提供瞬时电流
- 类比：芯片旁边的小水杯，随时能喝一口

InkTime PCB 上的电容配置：
```
电池 → [4.7µF] → TP4056 充电 → [10µF] → LDO 稳压 → [100nF] → ESP32
                                                      [1µF]
```

#### 2.3 LDO 稳压器

LDO (Low Dropout Regulator) = 低压差线性稳压器

```
输入: 3.7V (锂电池)  →  MIC5219  →  输出: 稳定的 3.3V
                         ↓
                    压差仅 0.4V
```

为什么不直接用电池供电？因为锂电池电压在 3.0V（没电）到 4.2V（满电）之间波动，ESP32 需要稳定的 3.3V。

#### 2.4 SPI 通信协议

SPI 是最简单的串行通信之一，只有 4 根核心线：

```
SCLK (时钟):  ┌─┐ ┌─┐ ┌─┐ ┌─┐ ┌─┐ ┌─┐ ┌─┐ ┌─┐
              ─┘ └─┘ └─┘ └─┘ └─┘ └─┘ └─┘ └─┘ └─

DIN (数据):   ─1───0───1───1───0───0───1───0──
                ↑   ↑   ↑   ↑   ↑   ↑   ↑   ↑
              每个时钟上升沿，屏幕读取 1 bit

CS (片选):    ──┐                               ┌──
                └───────────────────────────────┘
              低电平期间，屏幕才接收数据

DC (命令/数据): 低=发送命令  高=发送图像数据
```

一帧图像的数据量：800 × 480 像素 × 4bit/像素 = 192,000 字节

#### 2.5 上拉/下拉电阻

```
3.3V ──[10kΩ]──┬── GPIO (EN/RST)
               │
             [按键]
               │
              GND
```

- 按键没按下时：GPIO 通过 10kΩ 电阻连到 3.3V → 读到高电平
- 按键按下时：GPIO 直接连到 GND → 读到低电平
- 没有这个电阻，GPIO 在按键松开时会"悬空"，读到随机值

## 阶段 3：画原理图（第 5-6 周）

### 目标
用立创 EDA 画出完整的原理图。

### 工具
**立创 EDA**（https://lceda.cn）：免费、中文、在线使用、和嘉立创打板无缝衔接。

### 原理图模块划分

你的 PCB 需要包含以下 5 个模块：

```
┌─────────────────────────────────────────────────────┐
│                    完整 PCB 原理图                     │
│                                                      │
│  ① USB-C 充电接口                                     │
│     └→ ② TP4056 锂电池充电管理                        │
│           └→ ③ MIC5219 LDO 3.3V 稳压                 │
│                 └→ ④ ESP32-S3-WROOM-1 主控模块        │
│                       ├→ SPI 信号 → ⑤ 墨水屏 FPC 接口 │
│                       ├→ BOOT 按键                    │
│                       ├→ RESET 按键                   │
│                       └→ WiFi 重置按键                 │
│                                                      │
│  + 滤波电容（分布在各模块电源引脚旁）                    │
└─────────────────────────────────────────────────────┘
```

### 元件清单（BOM）

| 元件 | 型号 | 封装 | 数量 | 作用 |
|------|------|------|------|------|
| ESP32 模块 | ESP32-S3-WROOM-1-N8R8 | SMD | 1 | 主控 |
| LDO 稳压器 | MIC5219-3.3YM5-TR | SOT-23-5 | 1 | 3.3V 稳压 |
| 充电模块 | TP4056 | 模块 | 1 | 锂电池充电 |
| 电容 4.7µF | CL10A475KO8NNNC | 0603 | 1 | 输入滤波 |
| 电容 10µF | CL10A106KP8NNNC | 0603 | 1 | 输出滤波 |
| 电容 100nF | CL10B104KB8NNNC | 0603 | 1 | 去耦 |
| 电容 1µF | CL10A105KB8NNNC | 0603 | 1 | 去耦 |
| 电阻 10kΩ | 0603 封装 | 0603 | 2 | 上拉电阻 |
| 轻触按键 | 6×6×5mm | SMD | 2-4 | 复位/功能键 |
| FPC 连接器 | 8pin 或排针 | — | 1 | 连接墨水屏 |

4. 添加电源符号（VCC、GND）
5. 运行 ERC（电气规则检查），修复错误

## 阶段 4：PCB 布线（第 7-8 周）

### 目标
把原理图转换为实际的 PCB 布局。

### 关键布线原则

#### 4.1 元件摆放
```
优先级：
1. ESP32 模块放中间（它连接最多）
2. 滤波电容紧贴 ESP32 电源引脚（越近越好，< 3mm）
3. LDO 稳压器靠近 ESP32
4. FPC 连接器放板边（方便接排线）
5. USB 接口放板边（方便插线）
6. 按键放板边（方便按）
```

#### 4.2 走线规则

| 规则 | 说明 |
|------|------|
| 电源线宽 ≥ 0.5mm | VCC 和 GND 要粗，承载电流大 |
| 信号线宽 0.25mm | SPI 等信号线，电流小 |
| 拐弯用 45° | 不要 90° 直角，减少信号反射 |
| SPI 走线尽量等长 | SCLK、DIN、CS 长度差异 < 5mm |
| GND 铺铜 | 空白区域全部铺 GND 铜皮 |
| 电容就近放置 | 去耦电容到芯片引脚 < 3mm |

#### 4.3 板子尺寸建议
- 参考 InkTime PCB 尺寸，或根据你的外壳设计
- 2 层板即可（顶层走信号，底层铺 GND）
- 板厚 1.6mm（标准）

### 布线步骤

1. 从原理图导入 PCB 编辑器
2. 设置板框大小
3. 摆放元件（按上面的优先级）
4. 手动布线（先电源，再信号）
5. 铺铜（GND）
6. 运行 DRC（设计规则检查），修复错误

## 阶段 5：打板、焊接、调试（第 9-10 周）

### 5.1 打板

1. 在立创 EDA 导出 Gerber 文件
2. 上传到嘉立创（https://www.jlc.com）下单
3. 选择参数：2 层板、1.6mm 板厚、绿色阻焊（默认即可）
4. 价格：5 片通常 ¥5-20，3-5 天到货

### 5.2 焊接

需要的工具：

| 工具 | 用途 | 参考价格 |
|------|------|---------|
| 恒温烙铁（936 型） | 焊接 0603 元件和排针 | ¥50-100 |
| 热风枪 | 焊接 ESP32 模块（SMD） | ¥100-200 |
| 焊锡丝（0.5mm） | 细头，适合小元件 | ¥15 |
| 助焊剂 | 帮助焊锡流动 | ¥10 |
| 镊子 | 夹取 0603 小元件 | ¥10 |
| 放大镜/显微镜 | 检查焊点 | ¥30-100 |

焊接顺序（从矮到高）：
1. 0603 电阻、电容（最矮）
2. LDO 稳压器（SOT-23-5）
3. ESP32 模块（用热风枪）
4. 按键
5. 连接器、排针

### 5.3 调试

焊完后的检查步骤：

1. **目视检查**：用放大镜看每个焊点，有没有虚焊、短路
2. **万用表测量**：
   - VCC 和 GND 之间不应短路（阻值应 > 1kΩ）
   - LDO 输出应为 3.3V（接上电池后测量）
3. **上电测试**：接电池，看 ESP32 的 LED 是否亮
4. **烧录固件**：通过 UART 或 USB 烧录 InkTime 固件
5. **功能验证**：连接墨水屏，验证显示功能

### 常见问题排查

| 现象 | 可能原因 | 解决方法 |
|------|---------|---------|
| 上电无反应 | LDO 焊接不良 | 重新焊接，检查方向 |
| ESP32 不启动 | EN 引脚没上拉 | 检查 10kΩ 上拉电阻 |
| 屏幕不显示 | SPI 接线错误 | 逐一检查 6 根信号线 |
| 屏幕花屏 | BUSY 信号未接 | 确认 BUSY 引脚连接正确 |
| WiFi 连不上 | 天线被铜皮遮挡 | ESP32 天线区域不要铺铜 |

## 进阶方向

当你完成了基础 PCB 后，可以继续探索：

- **低功耗优化**：参考 InkTime 固件的 Deep Sleep 设计，实现 < 1mA 待机
- **外壳设计**：用 Fusion 360 或 FreeCAD 设计 3D 打印外壳
- **多屏幕适配**：设计通用接口，支持不同尺寸的墨水屏
- **太阳能供电**：加入太阳能充电板，实现永久续航
- **SMT 贴片**：嘉立创提供 SMT 贴片服务，可以让工厂帮你焊接
