# 环境变量配置指南

本文档解释了 `analyze_photos_docker.py` 如何读取配置，以及如何正确设置环境变量。

## 1. 配置读取机制分析

### 1.1 代码分析

`analyze_photos_docker.py` 中的配置读取代码（第21-28行）：

```python
# 从环境变量读取配置
ROOT_DIR = Path(__file__).resolve().parent.parent.parent

# 要扫描的图片目录（默认从环境变量读取，或使用 /photos）
IMAGE_DIR = Path(os.environ.get("IMAGE_DIR", "/photos")).expanduser()
if not IMAGE_DIR.is_absolute():
    IMAGE_DIR = (ROOT_DIR / IMAGE_DIR).resolve()
```

**工作原理**：
1. **确定项目根目录**：`ROOT_DIR` 通过 `Path(__file__).resolve().parent.parent.parent` 向上三级目录获取
2. **读取环境变量**：使用 `os.environ.get("IMAGE_DIR", "/photos")` 读取环境变量，如果不存在则使用默认值 `/photos`
3. **处理路径**：
   - `expanduser()` 处理路径中的 `~` 符号，展开为用户主目录
   - 检查路径是否为绝对路径，如果不是，则与 `ROOT_DIR` 结合形成绝对路径

### 1.2 其他配置项

脚本还读取以下环境变量：

- `DB_PATH`：数据库路径，默认 `./photos.db`
- `API_URL`：VLM API 地址，默认 `http://host.docker.internal:1234/v1/chat/completions`
- `MODEL_NAME`：模型名称，默认 `qwen3-vl-32b-instruct`
- `API_KEY`：API 密钥，默认空字符串
- `BATCH_LIMIT`：批量处理限制，默认无限制
- `TIMEOUT`：请求超时时间，默认 600 秒
- `WORLD_CITIES_CSV`：城市数据库路径，默认 `./data/world_cities_zh.csv`
- `HOME_LAT`、`HOME_LON`、`HOME_RADIUS_KM`：家的位置和半径

## 2. `export $(grep -v '^#' .env | xargs)` 失败原因

### 2.1 可能的失败原因

1. **.env 文件不存在**：项目中可能没有 `.env` 文件
2. **文件格式问题**：`.env` 文件格式不正确
3. **权限问题**：没有读取 `.env` 文件的权限
4. **命令语法问题**：不同 shell 的语法差异

## 3. 正确设置环境变量的方法

### 3.1 方法一：使用 .env 文件

1. **创建 .env 文件**：

```bash
cat > /Users/meng/Documents/GitHub/InkTime/.env << EOF
# 图片目录
IMAGE_DIR=/path/to/your/photos

# 数据库路径
DB_PATH=./photos.db

# VLM API URL
API_URL=http://localhost:1234/v1/chat/completions

# 模型名称
MODEL_NAME=qwen3-vl-32b-instruct

# 城市数据库路径
WORLD_CITIES_CSV=./data/world_cities_zh.csv

# 家的位置
HOME_LAT=22.543096
HOME_LON=114.057865
HOME_RADIUS_KM=60.0
EOF
```

2. **加载环境变量**：

```bash
# Bash  shell
export $(grep -v '^#' /Users/meng/Documents/GitHub/InkTime/.env | xargs)

# Zsh  shell
set -a
source /Users/meng/Documents/GitHub/InkTime/.env
set +a
```

### 3.2 方法二：直接设置环境变量

```bash
# 设置单个环境变量
export IMAGE_DIR=/path/to/your/photos
export API_URL=http://localhost:1234/v1/chat/completions
export MODEL_NAME=qwen3-vl-32b-instruct

# 运行脚本
python src/analysis/analyze_photos_docker.py
```

### 3.3 方法三：使用脚本包装器

创建 `run_analysis.sh` 脚本：

```bash
#!/bin/bash

# 设置环境变量
export IMAGE_DIR=/path/to/your/photos
export DB_PATH=./photos.db
export API_URL=http://localhost:1234/v1/chat/completions
export MODEL_NAME=qwen3-vl-32b-instruct
export WORLD_CITIES_CSV=./data/world_cities_zh.csv
export HOME_LAT=22.543096
export HOME_LON=114.057865
export HOME_RADIUS_KM=60.0

# 运行分析脚本
python src/analysis/analyze_photos_docker.py
```

然后运行：

```bash
chmod +x run_analysis.sh
./run_analysis.sh
```

## 5. 常见问题及解决方案

### 5.1 环境变量未生效

**问题**：设置了环境变量但脚本仍使用默认值

**解决方案**：
- 确保在同一终端会话中设置环境变量并运行脚本
- 检查环境变量名称是否正确（区分大小写）
- 验证环境变量值是否包含特殊字符，需要正确转义

### 5.2 路径问题

**问题**：脚本无法找到图片或数据库文件

**解决方案**：
- 使用绝对路径设置 `IMAGE_DIR` 和 `DB_PATH`
- 确保路径存在且有读写权限
- 检查路径中的空格和特殊字符

### 5.3 API 连接失败

**问题**：无法连接到 VLM API

**解决方案**：
- 确保 LM Studio 服务正在运行
- 检查 `API_URL` 是否正确
- 验证网络连接和防火墙设置
