# InkTime 项目分析报告

## 项目概述

InkTime 是一个智能墨水屏电子相框项目，通过 AI 理解照片内容，自动选择"历史上的今天"中最值得回忆的照片，并在 7.3 寸四色墨水屏上展示。

## 核心架构（三大部分）

### 1. 照片分析模块（Python）

**核心文件：** analyze\_photos.py

**主要功能：**

- 扫描照片目录，递归查找所有图片文件
- 调用视觉大模型（VLM）分析每张照片
- 提取 EXIF 信息（拍摄时间、GPS、设备信息等）
- 生成照片描述、分类、评分和文案
- 将所有数据存储到 SQLite 数据库

**技术要点：**

- 使用 PIL 处理图片，支持 EXIF 方向纠正
- 图片缩放控制（VLM\_MAX\_LONG\_EDGE）以减少推理成本
- GPS 逆地理编码，将经纬度转换为城市名
- 支持断点续跑，已处理的照片不会重复分析
- NAS 掉盘守护机制，自动重挂载网络存储

**AI 提示词设计：**

- 画面描述：详细描述照片内容
- 照片类型：人物、风景、美食等分类
- 评分系统：值得回忆度（0-100）、美观度（0-100）
- 旁白文案：8-24 个汉字的灵光一现短句

### 2. 图片渲染模块（Python）

**核心文件：** render\_daily\_photo.py

**主要功能：**

- 从数据库选择"历史上的今天"的高分照片
- 渲染成 480x800 的墨水屏格式
- 添加文案、日期、地点信息
- 应用四色抖动算法（黑/白/红/黄）
- 输出 .bin 文件供 ESP32 直接显示

**选片逻辑：**

1. 以今天的月日为目标（如 12-02）
2. 在所有年份该月日的照片中筛选回忆度 > 阈值的候选
3. 随机选择多张（默认 5 张）
4. 如果没有符合条件的，往前回溯最多 365 天
5. 兜底：选择全局回忆度最高的照片

**渲染技术：**

- 图片铺满裁剪算法
- 中文自动换行
- Floyd-Steinberg 抖动算法实现四色量化
- 行优先二进制格式输出（1 字节/像素）

### 3. ESP32 硬件模块

**核心文件：** ink-display-7C-photo.ino

**硬件配置：**

- 主控：ESP32-S3-N8R8（必须带 PSRAM ≥ 384KB）
- 屏幕：7.3 寸四色墨水屏 EL073TS3（49-pin）
- 驱动库：GxEPD2（GxEPD2\_730c\_GDEY073D46）

**引脚定义：**

```cpp
PIN_EPD_BUSY = 14
PIN_EPD_RST  = 13
PIN_EPD_DC   = 12
PIN_EPD_CS   = 11
PIN_EPD_SCLK = 10
PIN_EPD_DIN  = 9
```

**核心功能：**

- WiFi 连接和配置（支持 AP 配网模式）
- NTP 时间同步
- 定时从服务器下载 .bin 文件
- 墨水屏刷新显示
- Deep Sleep 低功耗管理（待机电流 < 1mA）
- 支持手动唤醒（RESET 键）和 WiFi 重置

**低功耗设计：**

- 刷新后立即进入 Deep Sleep
- GPIO 保持优化，只保留必要引脚
- WiFi/BT 彻底关闭
- 每日定时唤醒，半年续航（5000mAh 电池）

### 4. Web 服务器（Python）

**核心文件：** server.py

**主要功能：**

- Flask Web 服务器，提供 ESP32 下载接口
- WebUI 照片库管理界面（可关闭）
- 支持照片预览、筛选、排序
- 模拟墨水屏渲染效果预览
- 纯展示页面，提供类似于实体电子相册的沉浸式浏览体验

**API 端点：**

- `/static/inktime/{key}/photo_{idx}.bin` - ESP32 下载
- `/review` - 照片库管理界面
- `/sim` - 墨水屏模拟器
- `/images/...` - 原图访问
- `/display` - 纯展示页面
- `/display/<int:photo_id>` - 指定照片的纯展示页面
- `/api/display/next` - 获取下一张照片
- `/api/display/prev` - 获取上一张照片
- `/api/md_list` - 获取所有存在的 MM-DD 列表
- `/api/random_day` - 获取随机一天

### 5. 纯展示页面（HTML/JavaScript/CSS）

**核心文件：**

- `templates/display.html` - 纯展示页面模板
- `static/css/display.css` - 纯展示页面样式
- `static/js/display.js` - 纯展示页面脚本

**主要功能：**

- 极简风格的纯展示界面，仅展示一张照片、一段文字、今日日期和地点信息
- 自动切换照片（默认10秒间隔）
- 手动切换照片（左右箭头、点击屏幕边缘、触摸滑动）
- 点击页面跳转到照片详情页查看更多信息
- 响应式设计，适配不同屏幕尺寸

**技术要点：**

- 使用 Bootstrap 5 和 Font Awesome 提供基础样式和图标
- 深色背景，突出照片本身
- 平滑的过渡动画
- 支持触摸设备和键盘操作
- 模拟数据支持，无需后端 API 也能展示效果

**用户体验：**

- 沉浸式浏览体验，类似于实体电子相册
- 简洁明了的界面，无多余元素
- 流畅的照片切换效果
- 支持多种交互方式，适应不同设备

## 制作类似项目所需步骤

### 第一阶段：环境准备

1. **硬件采购**
   - ESP32-S3 开发板（带 PSRAM）
   - 7.3 寸四色墨水屏模块
   - 墨水屏转接板（49-pin）
   - 电池（可选，2 节 18650）
   - PCB 板和元件（如需自制）
2. **软件环境**
   - Python 3.10+
   - Arduino IDE
   - LM Studio 或其他 VLM 服务
   - exiftool（可选，用于 GPS 提取）

### 第二阶段：照片分析

1. **配置文件设置**
   - 复制 `config-example.py` 为 `config.py`
   - 设置照片库路径 `IMAGE_DIR`
   - 配置 VLM API 接口
   - 设置 DOWNLOAD\_KEY（安全密钥）
2. **运行分析**
   ```bash
   python3 analyze_photos.py
   ```
   - 扫描所有照片
   - 调用 AI 分析（需要较长时间）
   - 数据存储到 `photos.db`
3. **调整提示词**
   - 根据需要修改 AI 提示词
   - 调整评分标准和文案风格
   - 可以分批处理，支持断点续跑

### 第三阶段：图片渲染

1. **配置渲染参数**
   - 设置字体路径 `FONT_PATH`
   - 调整回忆度阈值 `MEMORY_THRESHOLD`
   - 设置每日照片数量 `DAILY_PHOTO_QUANTITY`
2. **运行渲染**
   ```bash
   python3 render_daily_photo.py
   ```
   - 自动选择"历史上的今天"照片
   - 渲染成 .bin 文件
   - 输出到 `output/inktime/` 目录
3. **预览效果**
   - 查看 `preview_*.png` 预览渲染效果
   - 调整布局和字体大小

### 第四阶段：ESP32 固件开发

1. **Arduino IDE 配置**
   - 安装 ESP32 Arduino Core
   - 安装 GxEPD2 库
   - 选择 ESP32-S3 开发板（开启 PSRAM）
2. **修改固件**
   - 同步 `DAILY_PHOTO_PATH_PREFIX` 与 `DOWNLOAD_KEY`
   - 根据实际硬件修改引脚定义
   - 调整屏幕尺寸参数
3. **编译烧录**
   - 连接 UART 串口
   - 编译并烧录固件
   - 通过串口监视器调试

### 第五阶段：服务器部署

1. **启动服务器**
   ```bash
   python3 server.py
   ```
2. **ESP32 配网**
   - 设备启动后自动进入 AP 模式
   - 连接 `InkTime-xxxx` 热点
   - 访问 `http://192.168.4.1/` 配置 WiFi 和服务器地址
3. **访问纯展示页面**
   - 打开浏览器访问 `http://服务器地址:5005/display`
   - 体验沉浸式照片浏览效果
   - 测试自动切换和手动切换功能
4. **定时任务（可选）**
   - 设置 crontab 每天自动渲染
   - 配置 systemd 服务自动启动

### 第六阶段：优化调试

1. **性能优化**
   - 调整 VLM\_MAX\_LONG\_EDGE 减少推理时间
   - 优化数据库查询
   - 减少不必要的网络请求
2. **功耗优化**
   - 调整刷新频率
   - 优化 Deep Sleep 时间
   - 检查 GPIO 保持设置
3. **效果优化**
   - 调整抖动算法参数
   - 优化文案生成质量
   - 改进选片逻辑

## 关键技术点

### AI 视觉理解

- 使用多模态大模型（如 Qwen3-VL）
- Base64 编码图片传输
- 结构化输出（JSON 格式）

### 图像处理

- PIL 图片处理和 EXIF 纠正
- Floyd-Steinberg 抖动算法
- 四色调色板量化

### 嵌入式开发

- ESP32 Deep Sleep 低功耗
- PSRAM 大内存管理
- 墨水屏驱动和刷新

### 数据库设计

- SQLite 存储照片元数据
- JSON 字段存储 EXIF 和 AI 结果
- 索引优化查询性能

## 项目特色

1. **智能选片**：不是随机播放，而是基于 AI 评分的"历史上的今天"
2. **低功耗**：Deep Sleep 待机电流 < 1mA，半年续航
3. **离线友好**：GPS 逆地理编码使用本地数据库
4. **断点续跑**：照片分析支持中断后继续
5. **灵活配置**：丰富的配置选项，适应不同需求
6. **纯展示页面**：提供类似于实体电子相册的沉浸式浏览体验
7. **多端体验**：同时支持墨水屏硬件和网页端浏览
8. **响应式设计**：适配不同屏幕尺寸，从手机到桌面设备

## 成本估算

- 硬件成本：ESP32-S3（\~50元）+ 墨水屏（\~200元）+ 其他元件（\~50元）≈ 300元
- 时间成本：照片分析（取决于数量，可能数小时）+ 调试（1-2天）
- 运行成本：本地 VLM 推理（免费），云端 API（按需付费）

## 总结

InkTime 是一个完整的端到端项目，融合了 AI、嵌入式、图像处理、Web 开发等多个技术领域。制作类似项目需要：

1. **硬件基础**：ESP32 开发和墨水屏驱动
2. **AI 能力**：视觉大模型调用和提示词设计
3. **图像处理**：抖动算法和色彩量化
4. **全栈开发**：Python 后端和 Web 前端
5. **系统集成**：各模块协同工作和调试
6. **前端开发**：响应式设计和用户体验优化

项目的核心价值在于用 AI 理解照片内容，实现智能选片，而不是简单的随机播放。通过添加纯展示页面，项目进一步扩展了用户体验，提供了类似于实体电子相册的沉浸式浏览方式，同时保留了墨水屏硬件的低功耗优势。

现在，用户可以通过两种方式欣赏照片：

- **墨水屏硬件**：低功耗、无需电源，适合放在桌面或床头
- **纯展示页面**：随时随地通过浏览器访问，支持多种交互方式

这种多端体验的设计使得 InkTime 不仅是一个硬件项目，更是一个完整的照片管理和展示系统，能够真正"拉回相册里的记忆"，为用户带来更加丰富和便捷的回忆体验。
