# 飞书文档操作规范

## 文档创建
- 文档语言默认中文
- 正文开头不要重复文档标题（飞书会自动显示标题）
- 创建后立即用 append 模式分段写入内容，避免单次写入过长导致超时

## 内容风格
- 主动使用飞书扩展语法增强可读性：
  - `<callout>` 高亮重要提示（💡蓝色=提示，⚠️黄色=警告，❌红色=危险，✅绿色=成功）
  - `<grid>` 分栏对比展示并列内容
  - 表格用标准 Markdown 语法
- 架构图、流程图使用 mermaid 代码块（飞书自动渲染为画板）
- 代码示例使用围栏式代码块并标注语言

## 分段写入策略
- 每次 append 控制在合理长度（约 2-3 个章节）
- 按逻辑章节拆分，不要在段落中间断开
- 写入完成后不需要再 fetch 验证

## 更新已有文档
- 小范围修改优先用 replace_range 或 insert_after，避免 overwrite 覆盖全文
- 定位内容时用 selection_with_ellipsis（开头内容...结尾内容）或 selection_by_title（## 章节标题）
- 修改表格/分栏等嵌套块时，精确定位到需要改的文本，避免破坏周围结构
